# Skillz Magic AI Studio Product Architecture

Skillz Magic AI Studio is designed as a professional AI automation asset factory. The app has three primary work zones: **Generate**, **Library**, and **Marketplace**. Generate creates Claude-compatible skills, prompts, and workflow blueprints from either detailed user instructions or smart defaults. Library stores official public references, uploaded user resources, and generated assets for later reuse. Marketplace packages generated assets as individual downloads or curated bundles with checkout-ready commerce flows.

| Zone | Primary User Job | Practical Output |
|---|---|---|
| Generate | Create usable Claude automation assets quickly | `SKILL.md`, prompt packs, workflow playbooks, manifests, usage guides |
| Library | Browse, import, search, and reuse resources | Official public references, uploaded files, locally saved generated assets |
| Marketplace | Prepare and sell assets | Product cards, bundle metadata, license text, downloadable zip packages, Stripe checkout session |

The autonomous mode uses a server-side LLM procedure so users can press one button and receive a complete asset even when they provide no inputs. Assisted mode uses the same procedure but adds role, industry, audience, constraints, monetization, and output type details for more tailored results. Client-side fallback templates remain available so the app stays useful if an external model call fails.

For commerce, the app creates Stripe Checkout Sessions server-side. The database stores only local business metadata and Stripe identifiers needed for fulfillment; Stripe remains the source of truth for payment status, amounts, customer billing details, and payment methods. Generated assets can be packaged as individual ZIP downloads or bundles directly in the browser, while purchase records are designed to support post-checkout fulfillment.

| Data Object | Storage Strategy | Reason |
|---|---|---|
| Generated asset draft | Browser local storage and optional server metadata | Keeps the workspace fast and private by default |
| Uploaded resource | Browser local library for the current implementation | Avoids unnecessary backend dependency for personal imports |
| Product listing | Database-ready metadata and client catalog state | Supports retail packaging and future admin publishing |
| Purchase record | Database table with Stripe session/payment IDs | Enables fulfillment without duplicating Stripe payment data |
| Download bundle | Client-generated ZIP package | Efficient for markdown/text asset delivery and avoids storing generated bytes unnecessarily |

The Claude-compatible plugin/skill export mode produces a folder-style ZIP package containing `SKILL.md`, `manifest.json`, `usage-guide.md`, prompt examples, and workflow notes. This does not claim to install directly into Claude on behalf of the user; instead, it creates a clean package that users can copy into a compatible Claude skill environment or use as Claude project instructions.
